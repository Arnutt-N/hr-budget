# Plan: Security Hardening — Legacy Remnants (post-cutover)

## Summary
Close three concrete security gaps on the **kept legacy PHP surface** that the Vue SPA did
not replace: (1) HTTP header / MIME injection in the legacy document-vault file download,
(2) the session cookie never sets `Secure` on HTTPS, and (3) the app emits **no** baseline
security response headers or Content-Security-Policy. All fixes are additive and bounded;
the `/api/v1/*` layer, `Jwt`, `AuthMiddleware`, and the API route block are untouched.

## User Story
As the HR division operating this app over HTTPS, I want downloads, session cookies, and
HTML responses to carry the standard browser security guarantees, so that a malicious or
malformed file name / MIME type cannot inject headers or be sniffed into stored XSS, the
session cookie cannot leak over plain HTTP, and clickjacking / content-injection vectors
are reduced.

## Problem → Solution
- `FileController::download()` interpolates `original_name` + `mime_type` straight into
  `Content-Disposition` / `Content-Type` headers (CRLF / quote injection, MIME sniffing).
  → sanitize via a small `Download` helper that mirrors the already-fixed API controller,
  add `X-Content-Type-Options: nosniff`, RFC 5987 `filename*` for Thai names, and a
  realpath containment check.
- `config/app.php > session.secure` defaults to `false`. → default to auto-detected HTTPS
  (env `SESSION_SECURE` still wins) so prod-over-HTTPS gets `Secure` without breaking
  Laragon HTTP dev.
- No CSP / `X-Frame-Options` / `nosniff` / `Referrer-Policy` / `Permissions-Policy`
  anywhere. → a `SecurityHeaders` Core class: **baseline** headers applied to every
  non-API response, plus a **strict CSP** applied only to the SPA shell (no inline scripts),
  while legacy server-rendered views keep baseline-only (they contain inline `<script>`).

## Step-by-Step Tasks

### Task 1: Create `src/Core/Download.php` (pure helpers + send wrapper)
- **ACTION**: New Core utility for safe file-download headers.
- **IMPLEMENT**:
  - `declare(strict_types=1);` + `namespace App\Core;`
  - `public static function safeFilename(string $name): string` — `str_replace(["\r","\n",'"'], '', basename($name))`; if result is empty, return `'download'`.
  - `public static function safeMimeType(?string $mime): string` — strip `\r\n`; trim; if it matches `^[A-Za-z0-9!#$&^_.+-]+/[A-Za-z0-9!#$&^_.+-]+$` return it, else `'application/octet-stream'`.
  - `public static function contentDisposition(string $name): string` — build `attachment; filename="<ascii-fallback>"; filename*=UTF-8''<rawurlencode(name)>` where ascii-fallback = `safeFilename()` with non-ASCII replaced by `_`. Use `rawurlencode($name)` for the `filename*` value.
  - `public static function sendFile(string $absPath, string $name, ?string $mime): void` — sets `Content-Type` (safeMimeType), `X-Content-Type-Options: nosniff`, `Content-Disposition` (contentDisposition), `Content-Length`, then `readfile(); exit;` (mirror API controller order).
- **MIRROR**: SAFE_DOWNLOAD pattern; Core static-class style from `Auth`.
- **IMPORTS**: none beyond core PHP.
- **GOTCHA**: `basename()` must run BEFORE encoding to strip any `../` path segments in the stored name. Do not call `header()` inside the pure methods — only inside `sendFile()`.
- **VALIDATE**: `php -l src/Core/Download.php`.

### Task 2: Create `src/Core/SecurityHeaders.php` (baseline + CSP)
- **ACTION**: New Core utility for security response headers.
- **IMPLEMENT**:
  - `declare(strict_types=1);` + `namespace App\Core;`
  - `public static function baselineHeaders(bool $isHttps): array` returns:
    - `'X-Content-Type-Options: nosniff'`
    - `'X-Frame-Options: DENY'`
    - `'Referrer-Policy: strict-origin-when-cross-origin'`
    - `'Permissions-Policy: camera=(), microphone=(), geolocation=()'`
    - if `$isHttps`: `'Strict-Transport-Security: max-age=31536000; includeSubDomains'`
  - `public static function spaCsp(): string` returns (single line, space-joined):
    `default-src 'self'; base-uri 'self'; object-src 'none'; frame-ancestors 'none'; form-action 'self'; img-src 'self' data:; font-src 'self' https://fonts.gstatic.com data:; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; script-src 'self'; connect-src 'self'`
  - `private static function isHttps(): bool` — `!empty($_SERVER['HTTPS']) && $_SERVER['HTTPS'] !== 'off'`
  - `public static function applyBaseline(): void` — loop `baselineHeaders(self::isHttps())` calling `header($h)`.
  - `public static function applySpaCsp(): void` — `applyBaseline()` + `header('Content-Security-Policy: ' . self::spaCsp())`.
- **MIRROR**: Core static-class style; `isHttps` detection mirrors `public/index.php:18`.
- **IMPORTS**: none.
- **GOTCHA**: `style-src` MUST keep `'unsafe-inline'` (PrimeVue/Vue inject runtime inline styles) and include `https://fonts.googleapis.com` (Google Fonts CSS) + `font-src https://fonts.gstatic.com` — confirmed from `public/app/index.html`. `script-src 'self'` is safe because the shell loads only an external module script.
- **VALIDATE**: `php -l src/Core/SecurityHeaders.php`.
